# Слияние конфигов — наследование профилей

> **Суть:** `LoadConfig` (`config.go:33`) собирает финальный конфиг из нескольких файлов
> по слоям, а профили моделей **наследуют поля из `model_definitions`** по принципу
> «inline-значение перекрывает, пустое — наследует».

## Слои загрузки (по порядку)
1. **Базовые `models.yaml`** из каждого searchPath (`.ai-review`/`.ai-reviewer`) →
   только `model_definitions` (`config.go:41`).
2. **Per-searchPath `config.yaml`**: ищется сначала `<dir>/<repo>/config.yaml`, затем
   `<dir>/config.yaml` (repo-scoped приоритетнее) (`config.go:67-80`). Рядом — sibling
   `models.yaml` override (`config.go:87`).
3. Все найденные конфиги **сливаются** (цикл по всем searchPath).

## Резолв профиля (наследование, `config.go:122-146`)
Для каждого `профиль → категория → ModelConfig`, если задан `id`:
```
def = ModelDefinitions[id]
если v.Provider == ""        → v.Provider = def.Provider
если v.Model == ""           → v.Model = def.Model
если v.MaxTokens == nil      → v.MaxTokens = def.MaxTokens
если v.ReasoningLevel == ""  → v.ReasoningLevel = def.ReasoningLevel
если v.InputPrice == 0       → наследует
если v.OutputPrice == 0      → наследует
```
→ профиль ссылается на definition по `id` и **точечно перекрывает** (напр. только
`reasoning_level: high`). Нет def → warning, частичный `v` остаётся.
Это и есть «позднее связывание» из [[Model Category и Profile — позднее связывание]].

## ⚠️ Разная семантика слияния по полям
| Поле | Семантика |
|---|---|
| `ModelDefinitions`, `ModelProfiles`, `PrimerTypes` | merge по ключу (last-wins) |
| `DefaultProfile`, `GlobalInstructions` | last non-empty wins |
| **`GlobalExcludes`** | **АППЕНД** — накапливаются со всех конфигов (`config.go:161`) |
Несимметрично: исключения копятся, инструкции — заменяются. Знать при отладке.

## Подвох — `0` как «наследовать» для цены
Цена — `float64` с сентинелом `0` (не указатель). → нельзя override-нуть цену в **0**,
если в def она ненулевая (унаследует def). Контраст с `MaxTokens *int` (где `nil` ≠ `0`,
см. [[Persona — корень агрегата ревью]]).

## Прочее
- Конфиг не найден → `os.ErrNotExist` (жёсткая ошибка).
- Precedence директорий: `<repo>/` > корень; `.ai-review` > `.ai-reviewer`.
- Тест precedence: [[Fixture dry-run — тест фильтрации без токенов]] (ассерт пути config).
