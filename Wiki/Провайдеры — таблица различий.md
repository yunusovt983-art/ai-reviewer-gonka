# Провайдеры — таблица различий

> **Суть:** три реализации [[Model Access — ACL над провайдерами|ModelClient]] ведут себя
> **по-разному** в двух критичных местах — enforcement JSON и учёт reasoning-токенов.
> **Anthropic — единственный, кто отстаёт по обоим.** Это скрыто внутри ACL.

## Сравнение

| Аспект | OpenAI | Anthropic | Gemini |
|---|---|---|---|
| **JSON-режим** | ✅ `ResponseFormat=JSONObject` (`:107`) | ❌ **игнор**, только промпт (`:163-171`) | ✅ `ResponseMIMEType=application/json` (`:292`) |
| **Reasoning-токены** | ✅ `CompletionTokensDetails.ReasoningTokens` (`:128`) | ❌ **всегда 0** (`:218,248`) | ✅ `ThoughtsTokenCount` (`:326`) |
| **reasoning_level →** | `ReasoningEffort` (`:117`) | thinking **budget** (расчёт `:192`) | `ThinkingLevel` low/med/high (`:300`) |
| **maxTokens=0** | без лимита (поле опускается) | **форс 65536** (API требует) (`:189`) | без лимита |
| **Транспорт** | non-stream `CreateChatCompletion` | **streaming** `NewStreaming` (`:217`) | non-stream `GenerateContent` |
| **Пустой ответ** | — | — | guard на 0 parts (`:331`) |

## Что общего (контракт ACL)
- `modelDisplay = model(reasoningLevel)` — суффикс срезается в
  [[Конструктор промптов — порядок и бюджет|tiktoken]].
- Фабрика `GetModelClient` (`models.go:360`): ключ из env, ошибка если пуст.
- `getEnv` (`models.go:21`): читает `KEYS.env` один раз (`sync.Once`), фолбэк `os.Getenv`.
- Все возвращают единый `ModelResult{Text, TokensIn/Out/Reasoning, FinishReason}`.

## ⚠️ Семантика токенов различается (тонко!)
Формула стоимости — `(out + reasoning)·outPrice` — предполагает, что reasoning **не
входит** в output. Это верно **не для всех**:
| Провайдер | reasoning внутри output? | следствие для формулы |
|---|---|---|
| **Gemini** | нет (`ThoughtsTokenCount` отдельно) | ✅ корректно |
| **Anthropic** | да (thinking в `output_tokens`); reasoning=0 | ✅ цена ок, но **не видно** thinking |
| **OpenAI** | да (`completion_tokens` включает reasoning) | ⚠️ вероятный **задвой** reasoning |
→ требуется **единый контракт `ModelResult`**: договориться, что `TokensOut` — output
*без* reasoning, а `TokensReasoning` — отдельно, и привести адаптеры к нему.
(Точные семантики стоит подтвердить по докам каждого SDK перед фиксом.)

## Почему это важно
1. Несогласованный учёт токенов → `total_cost` может врать (см. таблицу выше) и
   [[Учёт и отчётность — RunResults и stats]].
2. **Нормализация/pre-explainer на Anthropic** не имеют гарантии JSON — держатся на
   [[Finding — Value Object находки|extractJSON]] и тексте промпта.
3. Различия — аргумент за **контрактные тесты на каждый провайдер** при
   [[Рефакторинг к DDD-пакетам]] / [[PR-план рефакторинга — пошаговые diff'ы]].
