# 20 переносимых идей ai-reviewer

> **Суть:** что в `ai-reviewer` действительно умно и переносимо в свой инструмент.
> Distilled-список (стиль Карпати: сжать до сути). Каждая идея → ведёт к атомарной заметке.

## Архитектура и домен
1. **Ревью = оркестрация, а не один промпт.** Узкие персоны → выше сигнал.
   → [[Persona — корень агрегата ревью]]
2. **Repo-aware политика** — персоны/праймеры/вейверы версионируются с кодом.
   → [[Обнаружение артефактов — 3 слоя]]
3. **Стадии разной «дороговизны модели»** — токены по ценности шага.
   → [[Model Category и Profile — позднее связывание]]
4. **Позднее связывание модели** — категории, не имена; смена стека одним флагом.
   → [[Model Category и Profile — позднее связывание]]
5. **ACL над провайдерами** — `reasoning_level` нормализован поверх 3 API.
   → [[Model Access — ACL над провайдерами]]

## Контекст и стоимость
6. **Аннотированный дифф** `НОМЕР:±строка` — точные строки без галлюцинаций.
   → [[PRContext — ревьюируемый мир]]
7. **File-mode через фейковый дифф** — один код-путь для диффов и целых файлов.
   → [[PRContext — ревьюируемый мир]]
8. **Двухслойная фильтрация** — отбрасывай файлы до LLM (~95% экономии).
   → [[FilterSet — управление стоимостью]]
9. **Композитные фильтры `Any`/`All`** — булев DSL предикатов данными.
   → [[FilterSet — управление стоимостью]]
10. **`global_excludes` с override** — не жги токены на lock/generated/`go.sum`/`config.yaml`.
    → [[FilterSet — управление стоимостью]]

## Конвейер и качество
11. **Сырой текст → отдельная дешёвая нормализация в JSON** — раздели поиск и структуризацию.
    → [[Finding — Value Object находки]]
12. **Concepts** — семантический матчинг праймеров поверх путей.
    → [[Primer и Concept — инъекция знания]]
13. **Вейвер = фильтр + LLM-судья** + аудит `[Waived by..]`.
    → [[Waiver — LLM-судья подавления]]
14. **Атрибуция персон** через весь конвейер (`Finding.Source`, `@persona{ID}`).
    → [[Finding — Value Object находки]]
15. **Устойчивость к сбоям** — отчёт выдаётся всегда.
    → [[Sequence — конвейер ревью]]

## Инженерные приёмы
16. **Каскад `*int` лимита токенов** — `nil` ≠ `0` (не задано vs «без лимита»).
    → [[Persona — корень агрегата ревью]]
17. **Кэш pre-explainer'ов** по `SHA256(instructions+headSHA)` — кэшируй детерминированное.
    → [[Persona — корень агрегата ревью]]
18. **Двухшаговый парсинг frontmatter** — байтовый скан до YAML.
    → [[Обнаружение артефактов — 3 слоя]]
19. **`--dry-run` / `--context-eval` / `--prompt-only`** — наблюдаемость без затрат.
    → [[Composition Root — NewRunConfig]]
20. **Artifact-first observability** — полный paper trail (`runs/.../`, `run-log.jsonl`).
    → [[Sequence — конвейер ревью]]

## Как применить у себя (чек-лист)
- [ ] Разбить «универсальный ревью-промпт» на 3–5 узких персон.
- [ ] Положить правила ревью в репозиторий как версионируемые Markdown.
- [ ] Ввести `FilterSet`-предикат и фильтровать файлы *до* LLM.
- [ ] Разделить «поиск находок» (дорогая модель) и «нормализацию» (дешёвая).
- [ ] Писать артефакты каждого прогона на диск для аудита.
