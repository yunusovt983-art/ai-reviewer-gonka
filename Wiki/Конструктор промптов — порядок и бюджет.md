# Конструктор промптов — порядок и бюджет

> **Суть:** `PromptBuilder` (`context.go:806`) собирает промпт персоны из секций в
> **строго заданном порядке** и параллельно считает **токен-бюджет по секциям**
> (`PromptBreakdown`). Это и есть «центральная точка» промптинга из
> [[Контекстная карта — Bounded Contexts]].

## Точный порядок секций (verbatim-заголовки)
```
# PERSONA INSTRUCTIONS            ← всегда
---
# AGGREGATED REPORT               ← если IncludeFindings && есть summary (для post)
---
# PRIMER: <id> (Type: <t>)        ← по каждому совпавшему праймеру
**Type Intent:** <описание типа>     (из config.primer_types)
**Applies to:** <список файлов>
### Content: <тело праймера>
---
# PR METADATA  (Title / Description)
---
# CHANGED FILES                   ← список + inline "Explainer Analysis:" (по IncludeExplainers)
---
# UNIFIED DIFF: <file>            ← аннотированный дифф в ```diff```
   ИЛИ
# DIFF STATS                      ← если ExcludeDiff: N файлов, +X/-Y строк
---
# STANDARD INSTRUCTIONS           ← config.global_instructions
---
<PreRunExplainerSystemPrompt>     ← ТОЛЬКО для explainer/pre
```

## Идея — токен-бюджет как побочный продукт сборки
Каждая секция добавляется через `addEntry(category, subcategory, content)`, который
тут же считает `Chars` и `Tokens` → копит `PromptBreakdown{Entries, TotalChars, TotalTokens}`.
- `Build()` возвращает **и промпт, и breakdown**; `buildPrompt()` breakdown отбрасывает.
- Breakdown питает режим `--context-eval`/CSV → видно, **что именно** ест токены
  (персона? праймеры? дифф?). Связь: [[FilterSet — управление стоимостью]].

## Идея — точный подсчёт токенов через tiktoken (`CountTokens`)
```
model-specific encoding → fallback cl100k_base → fallback len/4
```
Имя модели чистится от суффикса `(...)` перед резолвом encoding (`context.go:838`).
→ оценка стоимости/контекста близка к реальной, а не «делим на 4».

## Идея — ExcludeDiff как «только статистика»
Если `persona.exclude_diff: true`, дифф заменяется строкой
`N files changed, +X lines added, -Y deleted` (счёт по аннотированному диффу,
`context.go:958`). Полезно персонам, которым важна **форма** изменения, а не строки —
дешевле по токенам.

## Идея — праймер несёт «Type Intent»
Тип праймера (`primer.Type`) разворачивается в человеко-читаемое
`**Type Intent:** <config.primer_types[type].Description>` — модель понимает *зачем*
этот праймер, а не только *что* в нём. См. [[Primer и Concept — инъекция знания]].
