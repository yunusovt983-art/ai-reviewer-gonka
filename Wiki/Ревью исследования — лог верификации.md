## 🔧 Уточнения, внесённые по итогам сверки
1. **Нормализацию делает `rc.FastestClient`** (не «какая-то дешёвая модель»),
   `persona.go:194`. Стоит в [[Finding — Value Object находки]].
2. **Нормализация логируется отдельной записью** `normalization:<personaID>` со своей
   ценой (`persona.go:212-224`) — в run-log на одну персону-ревьюера **две** записи.
3. **Агрегатор использует `Generate` (не `GenerateJSON`)** — вывод свободный Markdown,
   `pipeline.go:229`. JSON ждёт только normalize/pre-explainer.
4. **`--prompt-only` всё равно исполняет pre-explainer'ы** (`persona.go:115,124`) —
   от их JSON зависят промпты ревьюеров. Уточнено в [[Composition Root — NewRunConfig]].
5. **Severity-гайдлайны заданы дословно** + правило «**upgrade при согласии нескольких
   персон, downgrade при низком confidence**» (`pipeline.go:92-98,164-165`).
6. **`AggregateFindings` при 0 находок** возвращает готовый `"## Summary\nNo issues found"`
   без вызова модели (`pipeline.go:217`) — экономия.

## ⚠️ Найдено при копании — потенциальные дефекты upstream
1. **Несогласованная семантика reasoning-токенов между провайдерами** (`models.go`):
   формула `(out+reasoning)` верна для Gemini, у Anthropic reasoning=0 но thinking уже
   в `output_tokens` (цена ок, но **не видно** thinking), у OpenAI `completion_tokens`
   включает reasoning → **вероятный задвой**. Таблица: [[Провайдеры — таблица различий]].
   ⟵ *поправка прежней формулировки: это не «недосчёт Anthropic», а расхождение семантики;
   точные значения подтвердить по докам SDK.*
2. **`GenerateJSON` у Anthropic не форсит JSON** (`models.go:163-171`) — держится только
   на тексте промпта; для normalize/pre-explainer на Anthropic это хрупко (OpenAI/Gemini
   включают настоящий JSON-режим).
3. **Дублирование формулы стоимости ×3** (`settings.go:138`, `main.go:341`, +input-only
   в context-eval) — см. [[Отчёт и операбельность — report и finish-reason]].
(Это дефекты исследуемого репозитория, не кода пользователя — зафиксированы как знание.)
