# Целевая раскладка пакетов — диаграмма

> **Суть:** визуализация цели из [[Рефакторинг к DDD-пакетам]] и
> [[PR-план рефакторинга — пошаговые diff'ы]]. Правило: стрелки зависимостей идут
> **внутрь, к домену**; `domain` не импортирует ни `app`, ни `infra`.

## Граф зависимостей (mermaid)

```mermaid
flowchart TB
    cmd["cmd/ai-reviewer<br/><i>тонкий main: argv → app</i>"]

    subgraph app["internal/app — оркестрация"]
        root["config_root.go<br/>composition root"]
        planner["plan.go — Planner<br/>деление персон на группы"]
        runsvc["run.go — RunService<br/>стадии конвейера"]
    end

    subgraph domain["internal/domain — ЯДРО (без infra)"]
        subgraph review["review"]
            persona2["persona · finding"]
            primer2["primer · waiver · pipeline"]
            ports["ports.go<br/>ModelClient · PolicyStore"]
        end
        subgraph codebase["codebase"]
            diff2["diff · filterset · prcontext"]
        end
    end

    subgraph infra["internal/infra — адаптеры (реализуют port'ы)"]
        mc["modelclient<br/>openai · anthropic · gemini · Pool"]
        git2["gitsource — git/gh"]
        ps["policystore — scanner"]
        cs["configstore — config"]
        art["artifacts — OutputHandler"]
    end

    cmd --> app
    app --> domain
    app --> infra
    infra -.реализует.-> ports
    review --> codebase
    mc -.implements.-> ports
    ps -.implements.-> ports

    classDef core fill:#1f6f43,stroke:#0d3,color:#fff
    classDef adapter fill:#33415c,stroke:#5a7,color:#fff
    class review,codebase,persona2,primer2,ports,diff2 core
    class mc,git2,ps,cs,art adapter
```
